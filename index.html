Projects

Critter Auto-Battler

My partner and I drew inspiration from two of our favorite games: Pokémon and Super Auto Pets! My project partner and I are both avid fans of the Pokémon game series, and we were inspired by the simple yet strategic gameplay of simplistic auto battlers.

This project combines the typed battling of Pokémon with the easy-to-play fun of Super Auto Pets! The game uses Critter objects that attack each other and have 3 different statistics: health, attack, and kind (our version of type). Our game uses Pokémon-inspired type effectiveness to create an immersive, strategic experience!

Key features of the game:
The Critter class with health, attack, and type. Each one has a fun and unique name!
The Item class with temporary attack, armor, and kind to augment the Critters to your strategic advantage.
Team composition: Teams are made up of 3 critters that can hold a maximum of 1 item each to augment their stats.
Gameplay: We used a collide method that had the two leading critters attack one another until all critters are defeated.

Implementation: To implement this game, we used a state machine for each phase of the game.
Naming: Each player enters their name for the game.
Setup: Players pick their team and items from a list of randomly generated critters and items, adding variability for each round.
Team Preparation: Each player assigns items to their Critters and can switch their order if they desire.
Battle: Each players' teams collide until one team is defeated. Whoever's team is left standing wins the round! First to 5 points wins!
Game Over: This phase is reached once a winner is decided, and displays a victory message for the winner and closes the game.

This project was a rewarding opportunity to deepen my understanding of software development best practices. Using Git version control, my partner and I managed code changes efficiently, collaborated seamlessly, and maintained a clear project history. Through frequent commits and branching, we ensured a smooth workflow and resolved conflicts effectively. Working with a partner emphasized the importance of clear communication and division of responsibilities, fostering teamwork and accountability. The project also reinforced key object-oriented programming principles, which were applied to design flexible and modular classes for the game's Critters, Items, and overall mechanics. Throughout the project, we prioritized clean code practices, focusing on readability, modularity, and maintainability to ensure a high-quality codebase. By integrating concepts like dependency management, code reusability, and structured workflows, I gained valuable hands-on experience that reflects real-world development processes.
